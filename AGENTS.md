# Agent 开发指南

## C++20 编码要求

- 使用 C++20 标准
- 优先使用标准库，避免引入第三方依赖
- 使用 `std::optional` 表示可空值
- 使用 RAII 管理资源
- 使用 `constexpr` 和 `consteval` 适合的地方
- 智能指针管理动态生命周期
- 避免裸 `new`/`delete`

## 轻量化约束

- 不使用 Qt
- 不使用 Boost
- 不使用 Docker
- 不使用 Python 作为程序运行依赖
- 不开发图形界面或 Web 服务

## 不得虚构数据源规则

- 不得虚构外部网站的接口、URL、响应字段或数据
- 访问真实数据源前，必须先读取实际响应并保存测试样本
- 数据源访问失败时返回明确错误，保留数据库旧数据

## 模块分层要求

- 网络、数据解析、数据库和业务逻辑必须分离
- 上层业务只依赖接口，不依赖具体实现
- Parser 不得依赖网络
- HttpClient 不得解析 JSON 或业务数据

## 测试要求

- 新功能必须增加自动测试或可重复执行的验收方式
- 测试使用本地 fixture，不依赖公网
- 测试至少覆盖正常、异常、边界情况

## 提交要求

- Git 提交信息必须使用中文
- 每次只完成指定任务，不提前实现后续任务
- 完成后将提交推送到当前任务分支

## 提交日志更新要求

- 每次完成任务后，按格式追加记录到 提交日志.md

## 敏感信息保护要求

- 不记录 Token、Cookie 和敏感请求头
- 不关闭 HTTPS 证书校验
- 不提交包含密钥的本地配置文件
- .env 和包含密钥的本地配置文件必须被 gitignore
